- Course Title: Data Structures
- Course Code: B25CS03XX (confirm with program office)

## 1. Dual-Level Scope
- Awareness (A): Core ADTs, basic implementations, correctness, asymptotic reasoning, and common usage patterns required for placement readiness.
- Advanced (Adv): Design and analysis of advanced/engineered data structures, amortized analysis, performance trade-offs, and problem-focused engineering required for high CGPA and research/industry readiness.
- Assessment splits awareness and advanced outcomes: awareness items appear in all CIE/SEE components; advanced outcomes are assessed through higher-difficulty SEE questions, challenge assignments, and project tasks.

## 2. Course Outcomes (COs)
- CO1 (A): Describe fundamental data structures, their operations, and time/space complexity.
- CO2 (A): Implement arrays, linked lists, stacks, queues, trees, heaps, and hash tables with correct behavior and tests.
- CO3 (A→Adv): Analyze algorithmic complexity and select appropriate data structures for given constraints.
- CO4 (Adv): Design and implement balanced trees, efficient graph representations, priority queues, and hash-based structures for non-trivial problems.
- CO5 (Adv): Apply amortized analysis and design custom data structures addressing performance vs memory trade-offs.
- CO6 (A→Adv): Produce documented, testable solutions with clear API, complexity annotations, and unit tests.

## 3. Teaching-Learning Activities (14 weeks)
Week 1 — Introduction to ADTs; asymptotic notation; complexity reasoning
- Activity: Lecture + demo; activation exercises; formative quiz

Week 2 — Arrays & dynamic arrays
- Activity: Implementation lab; complexity analysis exercises

Week 3 — Linked lists (singly, doubly, circular)
- Activity: Pair-programming lab; debugging common pitfalls

Week 4 — Stacks & queues; applications (expression evaluation, BFS queue)
- Activity: Problem workshop; small assignment

Week 5 — Recursion; binary tree fundamentals; traversals
- Activity: Tracing exercises; mini-assignment

Week 6 — Binary Search Trees: operations, correctness, complexity
- Activity: Proof walkthrough; implementation exercise

Week 7 — Balanced trees (AVL): rotations and maintenance
- Activity: Rotation labs; challenge problems (Adv)

Week 8 — Heaps & priority queues; heapify, applications
- Activity: Implementation + benchmarking

Week 9 — Hashing: hash functions, collision resolution, performance
- Activity: Design experiment + benchmarking (Adv)

Week 10 — Graphs: adjacency list/matrix; BFS/DFS
- Activity: Implementation + visualization lab

Week 11 — Shortest paths & MST: Dijkstra, Bellman-Ford, Kruskal
- Activity: Algorithm tracing and coding (Adv)

Week 12 — Disjoint sets; segment trees; Fenwick trees (BIT)
- Activity: Problem-solving session (Adv)

Week 13 — Amortized analysis; memory layout and caching effects
- Activity: Case studies and measurement lab (Adv)

Week 14 — Revision; integrative mini-project presentations
- Activity: Student demos, peer review, feedback

## 4. Assessment Scheme
- Continuous Internal Evaluation (CIE) — 40% total
  - Quizzes (2–3): 15% (MCQ + short answers; awareness-focused)
  - Assignments / Programming tasks (2): 15% (one awareness, one advanced/challenge)
  - Midterm test: 10% (mixed)

- Semester End Examination (SEE) — 60% total
  - Paper structure: Section A (compulsory, awareness questions); Section B (choice, advanced problems).
  - Section A targets CO1–CO3; Section B targets CO4–CO6.

- Rubrics
  - Programming tasks: Correctness 60%, Efficiency/Complexity 20%, Tests/Docs 20%.
  - Written/SEE advanced problems: Algorithm correctness 50%, Analysis/Justification 30%, Design/Explanation 20%.

## 5. CGPA Calibration
- Mastery of Awareness-level COs (Section A + CIE awareness tasks) positions students in the mid CGPA band; strong performance on Section B and advanced assignments is required to reach ≥8 CGPA.

## 6. Sample SEE Question Paper (outline)
- Q1 (Awareness): Tree traversal / simple property counting — maps to CO2
- Q2 (Awareness): Analyze complexity of a code snippet — maps to CO1
- Q3 (Awareness): Hashing behavior / collision scenario reasoning — maps to CO2/CO3
- Q4 (Advanced): Design data structure for dynamic RMQ or range-update queries — maps to CO4/CO5
- Q5 (Advanced): Amortized analysis of dynamic array or splay-like operations — maps to CO5
- Q6 (Advanced): Graph design + justification for MST/SSP variant — maps to CO4
